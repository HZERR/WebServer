# WEB-приложения
Задача: Проведите анализ и опишите имеющихся на рынке программного обеспечения информационных систем, построенных по архитектуре WEB-приложений. Оцените и опишите возможности предлагаемых систем по архитектуре WEB-приложений и варианты их использования в компании.
Создайте с помощью Delphi 10.2 и MS Internet Information Server (IIS) приложение WEB-архитектуры на любую тему. Базу данных для WEB-приложения создать в MS SQL Server.

•	Ответом на задачу будет ссылка на репозиторий GitHub, где хранится Ваша программа. Или иным удобным для Вас способом.

•	Когда вы создаете базу данных в MySQL с помощью MySQL Workbench (или любого другого инструмента), вы можете экспортировать схему базы данных в виде скрипта SQL. Этот скрипт SQL содержит определения таблиц, связей, индексов и других структур базы данных, которые вы создали. Или иным удобным для Вас способом.

## Анализ рынка WEB-информационных систем

### 1. C# / ASP.NET Core + Angular
**Архитектура:**
- **Backend:** ASP.NET Core (REST API)
- **Frontend:** Angular (SPA)
- **БД:** PostgreSQL
- **Сервер:** IIS / Azure

**Возможности:**
- Чёткое разделение клиентской и серверной части.
- Высокая производительность и масштабируемость.
- Поддержка микросервисной архитектуры.
- Встроенные механизмы безопасности (JWT, OAuth2).

**Использование в компании:**
- Корпоративные порталы.
- CRM и ERP-системы.
- Системы электронного документооборота.
- Государственные и банковские WEB-сервисы.

---

### 2. Java (Spring Boot) + Angular
**Архитектура:**
- **Backend:** Spring Boot (REST / MVC)
- **Frontend:** Angular
- **БД:** PostgreSQL / Oracle / MongoDB
- **Сервер приложений:** Tomcat / WildFly

**Возможности:**
- Кроссплатформенность.
- Высокая надёжность и отказоустойчивость.
- Поддержка кластеризации и балансировки нагрузки.
- Широкое использование в enterprise-сегменте.

**Использование в компании:**
- Банковские и страховые системы.
- Крупные корпоративные WEB-платформы.
- Интеграционные шлюзы и API-сервисы.

---

### 3. CMS-системы (WordPress, Bitrix, Joomla)
**Архитектура:**
- **Серверная часть:** PHP
- **Клиент:** WEB-интерфейс (HTML, CSS, JavaScript)
- **БД:** MySQL / MariaDB
- **Сервер:** Apache / Nginx

**Возможности:**
- Быстрое создание WEB-сайтов и порталов.
- Большое количество готовых модулей и плагинов.
- Минимальные требования к разработке.
- Поддержка пользовательских ролей и контента.

**Использование в компании:**
- Корпоративные сайты.
- Информационные порталы.
- Интернет-магазины.
- Простые CRM и системы заявок.

**Недостатки:**
- Ограниченная масштабируемость.
- Зависимость от сторонних плагинов.
- Снижение безопасности при неправильной настройке.

---

### 4. Delphi + IIS (WEBBroker / WebModule)
**Архитектура:**
- **Backend:** Delphi (WEBBroker / ISAPI / CGI)
- **Сервер:** Microsoft IIS
- **БД:** MS SQL Server
- **Клиент:** WEB-браузер

**Возможности:**
- Высокая скорость выполнения серверного кода.
- Прямая работа с БД без промежуточных слоёв.
- Простота интеграции с существующими Windows-системами.
- Минимальные накладные расходы.

**Использование в компании:**
- Внутренние корпоративные WEB-приложения.
- Учётные системы.
- Legacy-проекты.
- Небольшие WEB-сервисы внутри организации.

**Недостатки:**
- Ограниченная популярность.
- Меньше готовых библиотек по сравнению с Java и C#.
- Используется в основном в учебных и специализированных проектах.

## Сравнительная характеристика

| Технология           | Масштабируемость | Скорость разработки | Тип использования                |
|----------------------|------------------|---------------------|----------------------------------|
| **ASP.NET + Angular** | Высокая          | Средняя             | Корпоративные ИС                 |
| **Java + Angular**    | Очень высокая    | Средняя             | Enterprise                       |
| **WordPress / CMS**   | Низкая–средняя   | Очень высокая       | Сайты, порталы                   |
| **Delphi + IIS**      | Низкая           | Средняя             | Внутренние системы               |

---

## Вывод

На рынке WEB-информационных систем доминируют решения на базе **C#** и **Java**, использующие **SPA**-фреймворки, такие как **Angular**, что обеспечивает высокую масштабируемость и производительность. **CMS-системы** (например, WordPress, Bitrix) часто применяются для быстрого создания WEB-порталов, но их масштабируемость ограничена. Решения на **Delphi + IIS** используются преимущественно для внутренних и учебных WEB-приложений с минимальными требованиями к масштабируемости.
